LAEQ-Corrélation Cartographique
Matching Tab

Ce plugin a pour objectif d'ajuster une trace GPS sur un réseau (map-matching) de façon semi-automatique
Il est recommandé de lire la documentation du plugin avant de l'utiliser: Lien

Description rapide du processus de map-matching

Le processus de map-matching passe par trois étapes successive:

Etape 1: Entrée

Cette étape prépare les données pour faciliter le travail des algorithmes de map-matching. Elle s'assure notamment de la topologie du réseau et de la validité des données en entrée.
Remplissez tous les champs, puis réduisez et corrigez le réseau de route afin d'atteindre l'étape suivante
Les champs à remplir sont les suivants :

Couche réseau:

  Cette liste déroulante présente toute les couches dont la géométrie est de type LineString actuellement actives dans votre projet QGIS. Sélectionnez la couche de route sur laquelle vous souhaitez ajuster votre trace GPS

Notez ici que les MultiLineString ne sont pas supportées. Il est possible de convertir une couche de MultiLineString en une couche de LineString avec l'outil "De morceaux multiples à morceaux uniques" de Qgis.

Couche Trace GPS:

  Cette liste déroulante présente toute les couches dont la géométrie est de type Point actuellement actives dans votre projet QGIS. Sélectionnez la couche de points que vous voulez matcher à un réseau de routes

OID (identifiant original) :

  Cette liste déroulante présente tous les attributs de la couche Trace GPS sélectionnée de type "Integer/Integer64/int/int8". Ce champ doit être un identifiant unique de tous les points et est utilisé pour ordonner les points de la couche. Un point présent plus tôt dans la trace doit avoir un indice plus petit qu'un point présent plus tard.

Taille du tampon:

  Ce slider représente la distance maximale à utiliser pour tracer un buffer autours des points de la trace GPS. Seules les lignes intersectant ce buffer seront considérées pendant le map-matching. Plus cette valeur est grande, plus le temps de calcul sera long par la suite.

Etape 2: Matching

Cette étape consiste à map-matcher les points de la trace GPS sur le réseau de route pré-traité de façon automatique. En cas d'erreur de la part de l'algorithme de map-matching, l'utilisateur peut modifier le trajet manuellement en sélectionnant les lignes réellement empruntées par la trace GPS et relancer l'algorithme sur cette nouvelle sélection.
Deux champs peuvent être renseignés :

Vitesse :

  Cette liste déroulante présente tous les attributs de la couche Trace GPS sélectionnée de type Real/double. Elle représente la vitesse de déplacement à chaque point de la trace. Il n'est nécessaire de renseigner ce champ qui si la méthode "matching par vitesse" est sélectionnée.

Boite algorithme de matching

  Cette liste déroulante permet de choisir le types de matching à effectuer. Trois options sont disponibles.

  • Matching par vitesse : Considérant la vitesse de déplacement, la trace GPS est découpée en sections en mouvement ou à l'arrêt. Tous les points d'une section à l'arrêt sont agrégés à leur localisation moyenne et snappés à leur localisation la plus proche sur la ligne sélectionnée sur le réseau. Les sections en mouvement sont ensuite ajustée entre les tronçons délimités par les sections à l'arrêt. Plus spécifiquement, les points sont ajustés le long du chemin parcouru en calculant le ratio entre la distance originale entre les points de la trace GPS d'une section et la longueur de cette section sur le réseau. L'objectif est ainsi de préserver l'écart naturel enregistré originalement entre les points.
  • Matching par distance : Cette approche propose d'ajuster les points le long du chemin sélectionné en calculant le ratio entre la longeur totale de la trace GPS et du chemin. Les points sont ensuite alignés successivement sur le chemin en reportant leurs distances successives originales multipliées par ce ratio.
  • Matching au plus proche : Cette approche propose simplement de projeter chaque point à sa ligne la plus proche sur le chemin sélectionné.

Problèmes potentiellement recontrés avec le plugin

Voici quelques conseil pour régler une partie des erreurs fréquemment rencontrées:

Les couches ne sont pas conforme aux standarts du plugin ou n'apparaissent pas

- Vérifier que les deux couches sont active (apparaissent bien dans le rendu)

- Vérifier le type de géométrie des couches sont bien Point et LineString (voir le menu propriété des couches). Il peut être nécessaire d'exporter une couche de lignes au format GPKG pour la forcer à être de type LineString

- Vérifier que les deux couches sont dans le même système de projection (SCR) cartésien (en mètre)

- Vérifier que la trace GPS possède un attribut OID de type integer

- Appuyer sur RELOAD pour afficher les couches ajoutées après le démarage du plugin

- Ne supprimez surtout pas de couche au cours des étapes au risque de devoir recommencer à 0
